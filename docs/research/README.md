# research/ -- Ideas and Explorations

This folder contains brainstorms, design explorations, and ideas that are **not yet part of the development plan**. Think of it as a scratchpad for concepts that may or may not make it into cfcf.

## Purpose

- Capture ideas early before they're forgotten
- Explore designs without committing to implementation
- Provide a place for "what if" thinking that doesn't belong in the formal design docs

## Lifecycle

1. An idea starts as a new `.md` file here
2. If it gains traction through discussion, it gets refined
3. When ready for implementation, it moves to `docs/design/` and gets added to `docs/plan.md`
4. If abandoned, it stays here as a record of what was considered and why it was dropped

## Naming

Use descriptive filenames: `multi-agent-communication.md`, `cost-tracking-approaches.md`, etc.
